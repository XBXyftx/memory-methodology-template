# Skill Lookup Guide

## Purpose

Use domain-specific skills when the task is framework-specific, tool-specific, or error-specific.

## Template

| Scenario | Recommended skill |
| -------- | ----------------- |
| Toolchain / environment | `<tool-skill>` |
| Project initialization | `<create-skill>` |
| API lookup | `<docs-skill>` |
| Syntax and language constraints | `<syntax-skill>` |
| Compile or type errors | `<error-skill>` |
| Runtime crash or faultlog | `<runtime-skill>` |
| UI behavior or layout | `<ui-skill>` |

## Rule

- Record which skill was used and why.
- Prefer the most specific guidance available.
- Do not substitute generic memory for specialized documentation when the task depends on exact framework behavior.
