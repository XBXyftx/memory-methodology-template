## Directory Roles

- `REQ-*` main documents are the entry points for a single requirement.
- Matching subdirectories hold original descriptions, screenshots, logs, investigations, review notes, and implementation records.
- `ANALYSIS-*` documents capture root-cause and architecture judgments.
- `SPEC-*` documents capture component or interface rules.

## Maintenance Rules

1. Create the entry document before adding deep details.
2. Keep entry files short and use child folders for long traces.
3. Prefer factual status words only.
4. Record what changed and what did not change.
5. Do not include real source code or confidential project facts.
