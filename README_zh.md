# 长期记忆方法论模板

一个可移植的渐进式索引文档脚手架，用于以可维护的方式沉淀项目知识、规则、需求、Bug 排查和操作经验。

> [English version](README.md)

## 这是什么

本模板提供了一套目录结构和文档规范，用于构建 AI 助手可高效导航的项目长期记忆。它与任何特定语言、框架或产品无关。

## 核心理念

- **渐进式索引** — 根索引 → 分区 README → 条目入口 → 子文档证据。每一层保持精简，细节向下沉淀。
- **每类条目一个入口** — 需求和 Bug 各自有唯一入口文件，配套同名子目录存放证据。
- **稳定规则与实时状态分离** — 通用的方法论和编写规则与临时交接快照分开维护。
- **证据驱动** — 结论基于调查记录，不凭提交标签推断。

## 目录结构

| 目录 | 用途 |
| --- | ------- |
| `00-index/` | 导航锚点，指向根索引 |
| `01-onboarding/` | 精简的会话启动上下文与约束 |
| `02-requirements/` | 需求、分析、方案设计、实施记录 |
| `03-api-practices/` | API 使用模式、类型约定、技术约束 |
| `04-operations/` | 操作历史与归档记录 |
| `05-reference/` | 项目概览、架构热点、技能指引、外部参考、经验教训 |
| `06-code-review-fixes/` | Code Review 发现的问题及修复记录 |
| `07-bug-reports/` | Bug 调查、修复方案、验证归档 |

## 根目录文件

| 文件 | 用途 |
| ---- | ------- |
| `MEMORY.md` | 全局导航索引与核心原则 |
| `rules.md` | 功能开发、Bug 修复、文档归档的必读规则 |
| `document-authoring-rules.md` | 记忆文档的命名、拆分、索引、链接、归档和审查策略 |
| `AI-handover.md` | 会话交接用的当前状态快照 |

## 使用方式

1. 将脚手架复制到你的项目工作区。
2. 将每个 `<placeholder>` 替换为具体的非敏感内容。
3. 创建或拆分文档时遵循 `document-authoring-rules.md` 中的规则。
4. 保持根索引和分区 README 同步更新。
5. 不要在此处存储密钥、私有路径或专有源码细节。

## 许可证

本模板作为起点提供。请根据你自己的工作流程和约束进行调整。
